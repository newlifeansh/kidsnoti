# 키즈 알림장 AI 서비스 화면 요구사항 명세서

## 문서 목적

이 문서는 키즈 알림장 AI 캘린더 & To-do MVP의 화면 요구사항을 정의한다.

서비스는 앱인토스(Apps in Toss) 스타일의 모바일 우선 UI를 기준으로 설계한다.  
사용자는 알림장 스크린샷을 업로드하고, AI가 추출한 일정과 To-do를 확인한 뒤 저장할 수 있어야 한다.

---

## 전체 UX 원칙

### 디자인 방향

- 모바일 단일 컬럼 UI를 기본으로 한다.
- Toss 스타일처럼 여백을 넉넉하게 사용한다.
- 화면당 핵심 CTA는 1개를 우선한다.
- 텍스트는 짧고 명확하게 작성한다.
- 복잡한 설명보다 카드, 배지, 상태값으로 이해시키는 구조를 사용한다.
- 사용자의 불안을 줄이기 위해 AI 결과는 항상 확인 후 저장한다.
- 자동 등록보다 “제가 이렇게 정리했어요. 확인해 주세요.” 흐름을 기본으로 한다.

### 톤앤매너

- 친근하지만 과하게 귀엽지 않게 작성한다.
- 부모가 빠르게 이해할 수 있는 문장을 사용한다.
- 기술 용어는 최대한 숨긴다.
- 예: “OCR 분석 중” 대신 “알림장을 읽고 있어요”
- 예: “Calendar API 저장 실패” 대신 “캘린더에 저장하지 못했어요”

### 기본 버튼 규칙

- Primary CTA는 화면 하단 고정 또는 주요 섹션 하단에 배치한다.
- 버튼 문구는 행동 중심으로 작성한다.
- 예:
  - “알림장 올리기”
  - “확인하고 저장하기”
  - “가족 초대하기”
  - “구글 캘린더 연결하기”

### 상태 표시 규칙

모든 비동기 작업에는 상태를 표시한다.

- 로딩
- 성공
- 실패
- 다시 시도 가능
- 사용자의 추가 확인 필요

---

## 공통 레이아웃

### 모바일 기준

- 최대 너비: 480px
- 전체 배경: 밝은 회색 또는 흰색
- 카드 배경: 흰색
- 카드 radius: 16px 이상
- 기본 padding: 20px 내외
- 하단 고정 CTA 영역 사용 가능

### 기본 화면 구조

```text
상단 영역
- 뒤로가기 또는 닫기
- 화면 제목
- 보조 설명

본문 영역
- 핵심 콘텐츠 카드
- 상태/목록/설정

하단 영역
- Primary CTA
- 필요 시 Secondary CTA
```

---

## 주요 사용자 흐름

```text
1. 진입
2. 로그인 또는 사용자 식별

```

# 1. 온보딩 화면

## 목적

서비스의 가치를 짧게 전달하고 사용자가 시작하도록 유도한다.

## 진입 조건

- 최초 방문 사용자
- 로그인하지 않은 사용자
- 가족 그룹이 없는 사용자

## 화면 구성

### 헤더

- 서비스명 또는 로고
- 짧은 소개 문구

예시 문구:

```text
알림장을 올리면
준비물과 일정을 알아서 정리해드려요
```

### 핵심 가치 카드

3개 정도로 제한한다.

```text
1. 알림장 사진을 읽어요
2. 준비물과 숙제를 To-do로 정리해요
3. 중요한 일정은 캘린더에 저장해요
```

### CTA

Primary:

```text
시작하기
```

Secondary, 선택:

```text
나중에 둘러보기
```

## 기능 요구사항

- 시작하기 클릭 시 로그인 또는 사용자 식별 단계로 이동한다.
- 이미 로그인된 사용자는 홈으로 이동한다.

# 2. 로그인 / 사용자 식별 화면

## 목적

사용자를 식별하고 개인 데이터를 안전하게 저장할 수 있게 한다.

## 앱인토스 환경

- 앱인토스 SDK 기반 사용자 식별을 우선 사용한다.
- 앱인토스 환경에서는 별도 Google 로그인 없이 진입할 수 있도록 설계한다.
- 로컬 개발 또는 외부 웹/PWA 버전에서는 Firebase Auth Google 로그인을 사용할 수 있다.

## 화면 구성

### 제목

```text
내 아이 알림장을 안전하게 관리해요
```

### 설명

```text
아이별 준비물, 일정, 알림 설정을 저장하려면 로그인이 필요해요.
```

### CTA

앱인토스 환경:

```text
계속하기
```

외부 웹/PWA 환경:

```text
Google로 계속하기
```

## 상태

- 로그인 전
- 로그인 중
- 로그인 실패
- 로그인 성공

## 에러 문구

```text
로그인하지 못했어요. 다시 시도해 주세요.
```

# 3. 홈 화면

## 목적

사용자가 오늘 챙겨야 할 준비물과 이번 주 일정을 빠르게 확인하게 한다.

## 화면 구성

### 상단 인사 영역

예시:

```text
오늘 챙길 준비물이 있어요
```

또는 To-do가 없을 때:

```text
오늘은 챙길 준비물이 없어요
```

### 아이 필터

- 전체
- 아이 이름별 탭
- 예: 전체 / 민준 / 서연

### 오늘 할 일 카드

표시 항목:

- To-do 제목
- 아이 이름
- 마감일
- 카테고리 배지
- 완료 체크 버튼

카테고리 예:

- 준비물
- 숙제
- 제출물
- 확인
- 납부
- 기타

### 내일/이번 주 일정 카드

표시 항목:

- 일정 제목
- 날짜
- 시간
- 아이 이름
- Google Calendar 저장 여부

### 빠른 CTA

Primary:

```text
알림장 올리기
```

Secondary:

```text
아이 관리
```

## 빈 상태

To-do와 일정이 모두 없을 때:

```text
아직 정리된 알림장이 없어요
알림장 사진을 올리면 준비물과 일정을 정리해드릴게요.
```

CTA:

```text
알림장 올리기
```

## 기능 요구사항

- To-do 완료 체크 가능
- 아이별 필터 적용
- 알림장 업로드 화면으로 이동
- 일정 카드 선택 시 상세 화면으로 이동
- To-do 카드 선택 시 상세 또는 수정 화면으로 이동

# 4. 아이 관리 화면

## 목적

사용자가 아이 프로필과 아이별 기본 알림 시간을 설정하게 한다.

## 화면 구성

### 제목

```text
아이 정보
```

### 아이 목록

각 아이 카드 표시 항목:

- 이름
- 학교/기관명
- 학년/반
- 기본 알림 시간
- 기본 캘린더, 연결된 경우

### CTA

Primary:

```text
아이 추가하기
```

## 아이 카드 액션

- 수정
- 삭제
- 알림 시간 변경
- 기본 캘린더 선택

## 빈 상태

```text
아직 등록된 아이가 없어요
아이를 등록하면 알림장 내용을 아이별로 정리할 수 있어요.
```

CTA:

```text
아이 등록하기
```

# 5. 아이 추가 / 수정 화면

## 목적

아이별 맞춤 분석과 알림 설정을 위해 아이 정보를 입력받는다.

## 입력 필드

필수:

- 아이 이름

선택:

- 학교/기관명
- 학년
- 반
- 기본 알림 시간
- 기본 캘린더

## 기본 알림 시간 UX

예시 옵션:

```text
전날 오후 8시
전날 오후 9시
당일 오전 7시
직접 설정
```

## CTA

추가 화면:

```text
아이 등록하기
```

수정 화면:

```text
저장하기
```

## 유효성 검사

- 아이 이름은 필수
- 아이 이름은 20자 이하 권장
- 알림 시간은 유효한 시간 형식이어야 함

## 에러 문구

```text
아이 이름을 입력해 주세요.
```

# 6. 가족 공유 화면

## 목적

같은 가족 구성원이 아이의 To-do와 일정을 함께 확인할 수 있게 한다.

## 화면 구성

### 제목

```text
가족과 함께 관리해요
```

### 설명

```text
초대 링크를 보내면 가족이 같은 준비물과 일정을 함께 볼 수 있어요.
```

### 가족 구성원 목록

표시 항목:

- 이름 또는 닉네임
- 역할
- 가입 상태

역할:

- owner
- member

사용자 표시 문구:

```text
관리자
가족 구성원
```

### CTA

Primary:

```text
초대 링크 만들기
```

Secondary:

```text
링크 복사하기
```

## 초대 링크 생성 후

표시:

```text
초대 링크가 만들어졌어요
가족에게 링크를 보내주세요.
```

액션:

- 링크 복사
- 공유하기
- 링크 만료하기, 선택 기능

## MVP 권한

owner:

- 아이 추가/수정
- 가족 초대
- 캘린더 연결 관리
- To-do 생성/수정/완료

member:

- To-do 보기
- To-do 완료 체크
- 일정 보기

# 7. 알림장 업로드 화면

## 목적

사용자가 알림장 스크린샷을 업로드하게 한다.

## 화면 구성

### 제목

```text
알림장 사진을 올려주세요
```

### 설명

```text
사진은 내용을 읽은 뒤 저장하지 않아요.
```

### 업로드 영역

- 이미지 선택 버튼
- 드래그 앤 드롭은 웹 데스크톱에서 선택
- 모바일에서는 사진 선택 또는 카메라 촬영 지원

### 지원 파일 안내

```text
JPG, PNG 이미지를 올릴 수 있어요.
```

### CTA

이미지 선택 전:

```text
사진 선택하기
```

이미지 선택 후:

```text
알림장 읽기
```

## 이미지 선택 후 미리보기

표시 항목:

- 선택한 이미지 썸네일
- 파일명, 선택 가능
- 다시 선택 버튼

## 기능 요구사항

- 이미지 용량 제한
- 이미지 형식 검사
- 업로드 전 미리보기
- 분석 요청 중 중복 클릭 방지
- 분석 후 이미지는 저장하지 않음

## 에러 문구

```text
이미지를 불러오지 못했어요.
다른 사진으로 다시 시도해 주세요.
```

```text
지원하지 않는 파일 형식이에요.
JPG 또는 PNG 파일을 올려주세요.
```

# 8. AI 분석 중 화면

## 목적

사용자에게 현재 처리 상태를 안심시키며 보여준다.

## 화면 구성

### 진행 상태 문구

단계별 문구 예시:

```text
알림장을 읽고 있어요
날짜와 준비물을 찾고 있어요
캘린더 일정과 할 일을 나누고 있어요
```

### 로딩 UI

- 부드러운 로딩 애니메이션
- 진행률은 실제 값이 없으면 표시하지 않는다
- 단계 문구는 순차적으로 변경 가능

### 취소 버튼

선택 기능:

```text
취소
```

## 실패 상태

OCR 실패:

```text
글자를 잘 읽지 못했어요
사진이 흐리거나 글자가 작으면 다시 찍어주세요.
```

LLM 분석 실패:

```text
내용을 정리하지 못했어요
잠시 후 다시 시도해 주세요.
```

CTA:

```text
다시 시도하기
```

Secondary:

```text
다른 사진 선택하기
```

# 9. 분석 결과 확인 화면

## 목적

AI가 정리한 결과를 사용자가 확인하고 수정한 뒤 저장하게 한다.

## 화면 구성

### 제목

```text
이렇게 정리했어요
```

### 보조 설명

```text
틀린 내용이 있으면 저장하기 전에 수정할 수 있어요.
```

### 섹션 1: 캘린더 일정

카드 표시 항목:

- 일정 제목
- 날짜
- 시간
- 아이 이름
- 장소, 있는 경우
- 신뢰도 낮음 표시, 필요한 경우
- Google Calendar 저장 예정 배지

빈 상태:

```text
캘린더에 넣을 일정은 찾지 못했어요.
```

### 섹션 2: To-do

카드 표시 항목:

- To-do 제목
- 카테고리
- 마감일
- 알림 예정 시간
- 아이 이름
- 신뢰도 낮음 표시, 필요한 경우

빈 상태:

```text
준비물이나 숙제는 찾지 못했어요.
```

### 섹션 3: 단순 안내

카드 표시 항목:

- 안내 제목
- 설명

접기/펼치기 가능

### 섹션 4: 원문 보기

- OCR 텍스트 표시
- 기본은 접혀 있음
- “원문 보기” 클릭 시 펼침

## 주요 CTA

Primary:

```text
확인하고 저장하기
```

Secondary:

```text
다시 분석하기
```

## 아이템 액션

각 카드에서 가능해야 하는 기능:

- 수정
- 삭제
- 일정/To-do 유형 변경
- 아이 선택 변경
- 날짜 변경
- 알림 시간 변경

## 신뢰도 낮은 항목 표시

조건:

- confidence가 일정 기준 미만
- 날짜 불확실
- 아이 불확실
- 조건부 문구 포함

표시 문구 예시:

```text
확인이 필요해요
```

상세 이유 예:

```text
날짜가 정확하지 않을 수 있어요.
```

## 저장 전 검증

- 일정 날짜가 없으면 저장 불가 또는 확인 필요
- To-do 제목이 없으면 저장 불가
- 아이가 없으면 “아이 미지정”으로 저장 가능하되 표시
- Google Calendar 연결이 없으면 Calendar 저장 대신 앱 일정으로만 저장하거나 연결 유도

분석 결과 카드의 세부 내용을 빠르게 수정할 수 있게 한다.

## 표시 방식

- 모바일에서는 바텀시트
- 데스크톱에서는 모달 가능

## 공통 필드

- 제목

# 19. 접근성 요구사항

- 모든 버튼은 명확한 텍스트 label을 가져야 한다.
- 색상만으로 상태를 구분하지 않는다.
- 중요 상태에는 텍스트 배지를 함께 표시한다.
- 터치 영역은 최소 44px 이상으로 한다.
- 폼 입력값에는 label을 제공한다.
- 로딩 상태는 스크린리더에 전달 가능해야 한다.
